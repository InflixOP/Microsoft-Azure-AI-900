# Microsoft-Azure-AI-900
The Azure AI-900 exam, or Microsoft Azure AI Fundamentals, is an entry-level certification that tests foundational knowledge of artificial intelligence and machine learning concepts and their implementation on Microsoft Azure. It covers topics like AI workloads, machine learning models, and Azure AI services etc. . 

# ☁️ ML Introduction

This repository provides a comprehensive introduction to Machine Learning (ML) and its various components. The content is segmented into different sections for ease of understanding.

## Layers of Machine Learning

Machine Learning (ML) is built upon various layers including data collection, data processing, model training, and model evaluation. These layers work together to create robust ML systems capable of learning from data and making predictions.

## Key Elements of AI

Artificial Intelligence (AI) encompasses various key elements such as algorithms, data, computation power, and human expertise. These elements are critical in developing intelligent systems that can mimic human behavior and decision-making processes.

## DataSets

Datasets are collections of data used to train and evaluate ML models. They are fundamental to the success of any ML project. Quality and quantity of data significantly impact model performance.

## Labeling

Labeling is the process of annotating data with relevant information. In supervised learning, labeled data is essential for training models to make accurate predictions.

## Supervised and Unsupervised Learning

- **Supervised Learning:** Uses labeled data to train models. The model learns to map input data to the correct output based on the provided labels.
- **Unsupervised Learning:** Uses unlabeled data. The model identifies patterns and structures in the data without any prior labeling.

## Reinforcement Learning

Reinforcement Learning (RL) is a type of ML where an agent learns to make decisions by performing actions in an environment to maximize cumulative rewards.

## Neural Networks and Deep Learning

Neural Networks are computational models inspired by the human brain. Deep Learning is a subset of ML that uses multi-layered neural networks to model complex patterns in data.

## GPU

Graphics Processing Units (GPUs) are specialized hardware accelerators that significantly speed up the training of deep learning models by parallelizing computations.

## CUDA

CUDA (Compute Unified Device Architecture) is a parallel computing platform and API model created by NVIDIA. It allows developers to use GPUs for general-purpose processing, significantly accelerating ML computations.

## Simple ML Pipeline

A typical ML pipeline involves the following steps:
1. Data Collection
2. Data Preprocessing
3. Model Training
4. Model Evaluation
5. Model Deployment

## Forecast vs Prediction

- **Forecast:** Typically refers to predicting future values based on historical data trends.
- **Prediction:** Generally refers to estimating an unknown outcome based on input data.

## Metrics

Metrics are used to evaluate the performance of ML models. Common metrics include accuracy, precision, recall, F1-score, and Mean Squared Error (MSE).

## Jupyter Notebooks

Jupyter Notebooks are interactive coding environments that allow developers to write, visualize, and share code. They are widely used in the ML community for experimentation and documentation.

## Regression

Regression analysis is used to model the relationship between a dependent variable and one or more independent variables. Common algorithms include Linear Regression and Polynomial Regression.

## Classification

Classification is the process of predicting the category or class of a given data point. Common algorithms include Logistic Regression, Decision Trees, and Support Vector Machines (SVM).

## Clustering

Clustering is an unsupervised learning technique used to group similar data points together. Common algorithms include K-Means, DBSCAN, and Hierarchical Clustering.

## Confusion Matrix

A Confusion Matrix is a tool used to visualize the performance of a classification model. It shows the true positive, true negative, false positive, and false negative counts, helping to evaluate model accuracy.

# ☁️ Common AI Workloads

## Anomaly Detection AI

Anomaly Detection involves identifying unusual patterns that do not conform to expected behavior. It is widely used in fraud detection, network security, and predictive maintenance.

## Computer Vision AI

Computer Vision is a field of AI that enables machines to interpret and make decisions based on visual data. Applications include image classification, object detection, and facial recognition.

## Natural Language Processing AI

Natural Language Processing (NLP) enables machines to understand, interpret, and generate human language. Applications include text analysis, language translation, and sentiment analysis.

## Conversational AI

Conversational AI refers to technologies that enable machines to engage in human-like conversations. Applications include chatbots, virtual assistants, and customer support automation.

Azure ML Studio is an integrated, end-to-end data science and advanced analytics solution. It enables data scientists and developers to build, deploy, and share predictive analytics solutions.

## Studio Compute

Studio Compute provides the computational resources required to run ML workloads. It includes virtual machines, GPU clusters, and other resources that can be scaled as needed.

## Studio Data Labeling

Data labeling in Azure ML Studio is a process to create labeled data for training machine learning models. It involves annotating data with tags that represent the features or outputs of interest.

## Data Stores

Data Stores in Azure ML Studio are storage abstractions that allow users to manage and access data from various sources such as Azure Blob Storage, Azure Data Lake, and more.

## Datasets

Datasets are organized collections of data that are used in experiments and training models. Azure ML Studio provides tools to create, manage, and share datasets.

## Experiments

Experiments in Azure ML Studio are runs of a machine learning script. They help in tracking the performance and results of different models and configurations.

## Pipelines

Pipelines in Azure ML Studio allow for the automation of workflows by connecting multiple steps into a single, repeatable process. They facilitate continuous integration and deployment of ML models.

## ML Designer

ML Designer in Azure ML Studio is a drag-and-drop interface for building machine learning models. It simplifies the creation of models by allowing users to visually compose and configure workflows.

## Model Registry

The Model Registry in Azure ML Studio is a centralized repository for storing and managing machine learning models. It helps in versioning, tracking, and deploying models efficiently.

## Endpoints

Endpoints are web services that host your machine learning models. They allow applications to interact with your models via RESTful APIs for inference.

## Notebooks

Notebooks in Azure ML Studio are interactive documents that combine live code, equations, visualizations, and narrative text. They are essential for data exploration and experimentation.

# ☁️ AutoML

## Introduction to AutoML

AutoML (Automated Machine Learning) automates the process of applying machine learning to real-world problems. It includes steps such as data preprocessing, feature selection, model training, and hyperparameter tuning.

## Data Guard Rails

Data Guard Rails ensure that the data used in AutoML processes meets certain quality and integrity standards. They help in preventing common data-related issues that can affect model performance.

## Automatic Featurization

Automatic Featurization in AutoML involves transforming raw data into features that can be used by machine learning algorithms. This step is crucial for improving model accuracy.

## Model Selection

Model Selection in AutoML involves choosing the best machine learning model from a set of candidates. AutoML evaluates multiple algorithms and selects the one that performs best on the given data.

## Explanation

Model Explanation in AutoML provides insights into how a model makes predictions. It helps in understanding the importance of different features and the overall behavior of the model.

## Primary Metrics

Primary Metrics are the main criteria used to evaluate the performance of a machine learning model. Common metrics include accuracy, precision, recall, F1-score, and Mean Squared Error (MSE).

## Validation Type

Validation Type refers to the method used to evaluate the performance of a machine learning model. Common validation techniques include cross-validation, train-test split, and bootstrapping.

# ☁️ Features of Generative AI Solutions

## AI vs Generative AI

AI refers to the broad field of creating intelligent systems that can perform tasks typically requiring human intelligence. Generative AI specifically focuses on creating new content, such as text, images, or music, based on learned patterns from existing data.

## What is a LLM (Large Language Model)

LLMs are advanced machine learning models that can understand and generate human language. They are trained on vast amounts of text data and can perform a wide range of natural language processing tasks.

## Transformer Models

Transformer models are a type of neural network architecture designed for handling sequential data. They are widely used in NLP tasks due to their ability to capture long-range dependencies and parallelize training.

## Tokenization

Tokenization is the process of converting text into smaller units called tokens. These tokens are the building blocks that models use to understand and generate language.

## Embeddings

Embeddings are vector representations of words or phrases that capture their meanings and relationships. They are used in various NLP tasks to improve model performance.

## Positional Encoding

Positional Encoding is used in transformer models to inject information about the position of tokens within a sequence. It helps the model understand the order and structure of the data.

## Attention

Attention mechanisms allow models to focus on specific parts of the input data when making predictions. They are crucial for improving the performance of transformer models in NLP tasks.
